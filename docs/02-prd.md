# 02 PRD

## MVP 目標
做出可用的 Flashcard 練習流程，讓使用者能快速開始、連續作答並看到即時結果。

## 核心流程
選擇練習模式 → 顯示題目 → 輸入答案 → 判斷正誤 → 顯示反饋 → 下一題 → 結束統計。

## 考試語境
本產品面向備戰升學考試的數學心算訓練，明確對應以下考試語境（文件與題目設計中均應具名，而非只寫「高考」）：

| 考試 | 說明 |
|------|------|
| **IAL** | International Advanced Level 數學 |
| **中國國內高考** | 全國／各卷高考數學運算節奏 |
| **香港 DSE** | 香港中學文憑試數學 |

題型與難度以這些考試中常見的心算、多步運算與基礎符號化簡為參照，不追求完整解題或證明。

## MVP 功能
- 題目生成：基礎四則、分數、小數、冪次，以及多步心算模板（見下方「題目形式」）；含少量概念型符號化簡題（見「符號化簡範圍」）。
- 答案輸入：支援鍵盤輸入與 Enter 送出。
- 即時反饋：顯示正確／錯誤、正確答案與用時。
- 練習統計：本輪題數、正確率、平均用時、錯題列表。
- **弱項分析**：依大題型（`type`）與細項能力標籤（`tags`）聚合本輪表現，標示偏弱項目（見「弱項診斷」）。
- **弱項專攻模式**（P1，與弱項分析同迭代）：依 localStorage 最近練習紀錄推導弱項，只出含目標 tag 的題目。
- **題量檔位**：低檔 10 題、中檔 20 題（預設）、高檔 50 題；各練習模式共用。
- 本地記錄：用 localStorage 保存最近練習與偏好設定。

## 題目形式

### 題型比例
- 填空題：約 80%。
- 選擇題：約 20%。

### 填空題
用戶需要自行輸入答案。

例如：
- `15 × 17 = ?`
- `√144 = ?`
- `log₂(32) = ?`

### 選擇題
提供 4 個選項。

要求：
- 只有 1 個正確答案。
- 干擾項需合理。
- 避免一眼看出答案。

例如：

`25 × 16 = ?`

- A. 350
- B. 375
- C. 400
- D. 425

### 多步心算模板
除單步四則外，MVP 優先引入**多重模板**（不同數字組合、非固定答案），避免使用者記住固定題目。每題可同時標記多個細項能力 tag，例如：

| 模板 | 診斷相關 tags（示例） |
|------|----------------------|
| `(a + b) × c = ?` | addition, multiplication, order-of-operations, working-memory |
| `(a × b) + c = ?` | multiplication, addition, order-of-operations, working-memory |
| `(a² - b²) = ?` 或 `(a + b)(a - b) = ?` | multiplication, subtraction, working-memory |
| `√((-n)²) = ?`（數值版） | square-root, absolute-value |
| `√(A²) = ?`（概念版，答案格式 `\|A\|`） | square-root, absolute-value, symbolic-simplification |

### 題目生成規則
- 題目數字盡量隨機。
- 盡量避免短時間內重複題目。
- 同一訓練回合內避免出現相同題目。
- 優先生成未出現過的題目。

### 使用者體驗
目標：
- 快速作答。
- 快速反饋。
- 保持流暢節奏。

避免：
- 大量重複題目。
- 題目過於固定。
- 記憶答案而非訓練計算能力。

## 符號化簡範圍

本產品在 MVP 階段**只做少量、固定格式的概念型符號化簡**，作為升學考試心算能力的延伸，**不做完整代數等價判定**。

### 納入範圍
- **數值版**：例如 `√((-7)²) = ?`，答案為具體數值 `7`。
- **概念版**：例如 `√(A²) = ?`，要求使用者輸入固定格式 `|A|`（絕對值符號包裹原代數）。

### 不納入範圍（本迭代非目標）
- 不判定 `abs(A)`、`±A` 等與 `|A|` 同義但格式不同的答案。
- 不做一般根式化簡、因式分解或任意代數式的等價比對。
- 不以「符號運算引擎」取代心算訓練；符號題僅作為概念檢查的補充。

## 弱項診斷

### 目標
讓使用者在完成一輪練習後，能同時從**大題型**（如 arithmetic、fractions、powers）與**細項能力**（如 multiplication、order-of-operations、absolute-value）兩個粒度，了解自己在速度與準確率上的偏弱處，並可一鍵啟動弱項專攻。

### 診斷依據
每題已保存 `question`、`isCorrect`、`timeMs`；題目 metadata 含 `type` 與 `tags[]`。弱項判斷**同時參考正確率與平均每題用時**，依難度使用固定門檻：

| 難度 | 正確率偏低 | 平均用時偏高 |
|------|-----------|-------------|
| easy | < 90% | > 5 秒 |
| medium | < 80% | > 8 秒 |
| hard | < 70% | > 12 秒 |

- 某 tag 或 type 的樣本題數 **少於 2 題**時，不標為「偏弱」，僅顯示「資料不足」。
- 本輪題數太少或無明顯弱項時，提示：「這輪樣本較少，先多練幾輪再看趨勢。」

### 結果頁呈現
在本輪統計下方顯示「偏弱項目」：
- 優先列出最需注意的大題型。
- 再列出 2–3 個細項能力標籤。
- 每項顯示正確率、平均用時、樣本題數，並附簡短診斷文案（例如：「乘法正確率不低，但平均用時偏高，建議練多步心算節奏。」）。

### 弱項專攻模式
- **弱項來源**：從 localStorage 最近 3–5 輪或最近 50 題 attempts 聚合，重用上述 breakdown 與門檻；優先取細項 tags 中標記為「偏弱」者（最多 3 個），不足時退回大題型 type。
- **出題**：`weakness-focused` 模式下，只從 `question.tags` 與 `targetTags` 有交集的模板中抽取；候選不足時可放寬至同一大題型，仍避免 session 內重複題目；完全無候選時 fallback 至 `mixed` 並提示。
- **歷史不足**：總題數 < 10 時回傳空集合，提示「先完成幾輪混合練習，再使用弱項專攻」。
- **入口**：結果頁每項弱項旁「專攻此項」、底部「專攻全部弱項」；模式選擇器亦提供「弱項專攻」；練習中顯示目前專攻標籤。

## 邊界與擴展
- 非目標：完整大題講解、課程、商業訂閱、班級管理、複雜社群；**完整代數等價判定**；長期跨 session 進步曲線圖表；**AI 出題**；弱項專攻的自適應難度或個人化門檻。
- 未來：章節題庫、難度分級、錯題複習、限時挑戰、進步曲線、PWA 離線使用。
