# AI Collaboration Rules

## 原則

- 不要假設需求
- 不要新增未確認功能
- 先分析再實作
- 每次修改先說明影響
- 保持文件同步更新

## 決策模式

當文件出現 `[TO_BE_DEFINED]` 時，依 [decision-assistant.md](./decision-assistant.md) 擔任**產品顧問**：

- 提出 3–5 個選項、分析利弊、評估成本與商業價值、給出推薦
- **不要**要求決策者自行填寫空白
- 決策者確認後，由 AI 更新文件

## 工作流程

1. 閱讀 docs
2. 提出方案（含 `[TO_BE_DEFINED]` 時走決策助理流程）
3. 等待確認
4. 實作
5. 更新文件
